## What's New in v2.0.0

### Orchestrator Integration
This release integrates the [CRE Acquisition Orchestrator](https://github.com/ahacker-1/cre-acquisition-orchestrator) by Avi Hacker -- a multi-agent pipeline engine for CRE acquisitions. The plugin's skills now serve as execution methodologies for the orchestrator's specialist agents, adding institutional-grade depth to every pipeline phase.

### Orchestration Engine (10 Pipelines)

The plugin now includes a self-contained orchestration engine with 10 multi-agent pipelines. Each pipeline coordinates skills and agents into phased workflows with checkpoints, verdict logic, and optional challenge layers.

Run any orchestrator with `/cre-skills:orchestrate {name}`.

| Pipeline | Phases | Skills Wired | Agents |
|---|---|---|---|
| acquisition | 6 (DD, UW, Financing, Legal, Closing, Challenge) | 15+ | 22 specialist + 8 challenge |
| capital-stack | 5 (Qualification, Sizing, Structuring, Optimization, IC) | 6 | 4 capital markets |
| hold-period | 7 (Onboarding, Monitoring, Leasing, Capital, Tenant, Reposition, Trigger) | 18 | 4 asset management |
| disposition | 7 (Hold/Sell, Pricing, Marketing, Buyers, Offers, DD Mgmt, Close) | 10 | 3 disposition |
| development | 6 (Land, Entitlement, Proforma, Construction, Lease-Up, Stabilization) | 8 | 5 development |
| fund-management | 6 (Formation, Raise, Deploy, Monitor, Distribute, Exit) | 12 | 3 fund operations |
| research | 5 (Macro, Submarket, Competitive, Opportunity, Memo) | 6 | 2 research |
| strategy | 5 (Capital, Cycle, Strategy, Portfolio, Memo) | 5 | 2 strategy |
| portfolio | 6 (Composition, Concentration, Attribution, Rebalance, Stress, Report) | 8 | 2 portfolio |
| lp-intelligence | 5 (GP Eval, Data Request, Performance, Portfolio, Re-Up) | 6 | 3 LP intelligence |

Derived from the [CRE Acquisition Orchestrator](https://github.com/ahacker-1/cre-acquisition-orchestrator) by Avi Hacker, adapted as a self-contained plugin layer. Includes 8 investor profiles, a strategy matrix, cross-chain handoff registry, disagreement and reversal-trigger schemas, and a 3-tier challenge layer with conditional triggers.

## Installation

### Upgrading from v1.0.0

```bash
cd /path/to/cre-skills-plugin
./scripts/update.sh
```

The update script preserves your `~/.cre-skills/` data (config, telemetry, feedback, brand guidelines).

### Fresh Install

```bash
# Option 1: Claude Code CLI (recommended)
claude plugin add github:mariourquia/cre-skills-plugin

# Option 2: One-line install
curl -fsSL https://raw.githubusercontent.com/mariourquia/cre-skills-plugin/main/scripts/install.sh | bash

# Option 3: Manual
git clone https://github.com/mariourquia/cre-skills-plugin.git
claude plugin add ./cre-skills-plugin

# Option 4: macOS DMG
# Download cre-skills-v2.0.0.dmg from the assets below
```

### Verify Installation

```bash
./scripts/verify-install.sh
```

## Workflow Chains (6 End-to-End Pipelines)

1. **Acquisition Pipeline**: sourcing -> screen -> OM reverse -> rent-roll -> underwriting -> sensitivity -> IC memo -> LOI -> PSA -> DD -> close
2. **Capital Stack Assembly**: underwriting -> loan sizing -> mezz/pref -> JV waterfall -> cap stack optimizer -> refi decision
3. **Hold Period Management**: budget -> performance dashboard -> capex / lease compliance / delinquency / retention -> NOI sprint
4. **Disposition Pipeline**: performance -> disposition strategy -> [SELL] prep + 1031 | [HOLD] refi | [REFI] loan sizing
5. **Development Pipeline**: land residual + entitlement -> dev proforma -> construction budget -> loan sizing -> cap stack -> IC memo -> lease-up -> refi
6. **Fund Management**: formation -> pitch deck -> capital raise -> portfolio allocator -> [deploy via acquisition] -> quarterly update + attribution

## Infrastructure & Quality

### Test Suite
9 structural integrity tests covering plugin JSON validity, skill file presence (98 skills), hook and calculator syntax, required root files, agents index, routing index, and skill count verification. Run with `python3 -m pytest tests/ -v`.

### CI/CD
GitHub Actions workflow testing across Python 3.10/3.11/3.12 and Node 18/20/22 (9-cell matrix). Runs on push to main and all PRs.

### Cross-Platform Install Scripts
- `scripts/install.sh` -- fresh install with v1->v2 migration, prerequisite checking, user data preservation

- `scripts/uninstall.sh` -- clean removal with optional data preservation
