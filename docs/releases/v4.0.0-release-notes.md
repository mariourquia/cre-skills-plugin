## What's New in v4.0.0

28 commits since v3.0.0. This release consolidates v4.0.1 and v4.1.0 into a single v4.0.0.

---

### Catalog-Driven Architecture

The plugin now runs on a **canonical catalog** (`catalog/catalog.yaml`, 203 items). Every public surface -- README stats, plugin.json description, hooks prompt, routing table, registry.yaml -- is generated from the catalog. CI catches drift automatically.

**Catalog-driven router**: `routing/skill-dispatcher.mjs` reads `dist/catalog.json` with artifact-aware matching (`--artifact "OM"`), confidence scoring, stable-first ordering, and markdown fallback.

### MCP Server -- Claude Desktop Support

Zero-dependency MCP server (`mcp-server.mjs`) that makes all 112 skills accessible from Claude Desktop, not just Claude Code.

**8 MCP tools:**

| Tool | Purpose |
|------|---------|
| `cre_route` | Route any CRE query to the right specialist skill |
| `cre_list_skills` | Browse and filter the 112-skill catalog |
| `cre_skill_detail` | Read the full process doc for any skill |
| `cre_workspace_create` | Start tracking a deal, asset, or fund |
| `cre_workspace_get` | Resume a workspace from a prior session |
| `cre_workspace_list` | See all active workspaces |
| `cre_workspace_update` | Add facts, decisions, next actions |
| `cre_send_feedback` | Submit feedback or bug reports |

**Manual setup** (if not using an installer):
```json
{
  "mcpServers": {
    "cre-skills": {
      "command": "node",
      "args": ["/path/to/cre-skills-plugin/mcp-server.mjs"]
    }
  }
}
```

### 7 Workspace Skills

Top-level entrypoints that route to specialist skills internally:

| Workspace | Purpose |
|-----------|---------|
| `deal-intake` | Full deal evaluation pipeline |
| `lease-strategy-papering` | Leasing workflows and negotiations |
| `asset-ops-cockpit` | Property management and performance |
| `capital-projects-development` | Ground-up development and construction |
| `fund-lp-reporting` | Fund management and LP communications |
| `navigator` | Meta-router: classifies and routes any CRE request |
| `plugin-admin` | Configuration, diagnostics, workspace management |

Persistent workspace state (local JSON at `~/.cre-skills/workspaces/`) enables cross-session continuity for deals, assets, and funds.

### Feedback Retry Outbox

Failed remote feedback submissions are now queued in `~/.cre-skills/outbox.jsonl` and automatically retried on next session start. 4-second timeout per request, max 5 attempts before permanent eviction. Fire-and-forget -- never blocks session start.

### Telemetry (Opt-Out)

Enabled by default. Records only: skill slug, date, anonymous UUID. No deal data, financials, prompts, or identity. All local at `~/.cre-skills/telemetry.jsonl`.

To opt out: `"telemetry": false` in `~/.cre-skills/config.json`

### Feedback (Ask Each Time)

When submitting feedback or bug reports, you are asked before anything is sent remotely. Includes 30-day skill usage summary for diagnostic context.

### Also in this release

- **Agent validation fix** -- `name` field added to all 54 agents, flattened from 7 subdirectories to flat `agents/` root for Desktop/Cowork compatibility
- **Windows installer fix** -- registers in `~/.claude/plugins/` (was writing to wrong directory)
- **5 output styles** -- `exec-brief`, `ic-memo`, `pm-action-list`, `lender-brief`, `lp-update`
- **Installer CI gates** -- smoke tests, release gating, version truth enforcement
- **GitHub Actions** bumped to v6/v7 (Node.js 20 deprecation)
